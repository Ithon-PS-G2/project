# Requisitos Não Funcionais

## O que são?

**Requisitos não funcionais** são especificações que descrevem como um sistema deve funcionar, em vez de o que ele deve fazer. Eles definem características de qualidade, desempenho, restrições e padrões que o sistema precisa atender.

Esses requisitos são importantes porque influenciam diretamente **a qualidade e a experiência de uso do software**, além de impactarem arquitetura, infraestrutura e custos do projeto.

No Sistema Simplificado de Gestão Familiar, os requisitos não funcionais garantem que a aplicação seja segura, fácil de usar, rápida, compatível com diferentes dispositivos e preparada para futuras expansões. Esses requisitos ajudam a assegurar uma boa experiência para os usuários e maior confiabilidade no gerenciamento das informações financeiras e patrimoniais.

---

## Segurança

1. O sistema deve exigir autenticação por login e senha para acesso às informações.
2. Os dados devem ser armazenados de acordo com a LGPD.

---

## Usabilidade

1. A interface deve ser intuitiva e permitir fácil navegação.
2. O sistema deve ser responsivo, funcionando corretamente em celulares, tablets e computadores.

---

## Performance

1. As páginas principais devem carregar em até 3 segundos.
2. Os dashboards financeiros mensais devem ser gerados em no máximo 5 segundos.

---

## Escalabilidade

1. O sistema deve permitir a adição de novos módulos futuramente sem grandes alterações na estrutura.
2. A aplicação deve suportar aumento gradual na quantidade de usuários e registros cadastrados.

---

## Compatibilidade

1. O sistema deve funcionar nos navegadores Google Chrome, Microsoft Edge e Mozilla Firefox.
2. O sistema deve ser acessível em sistemas operacionais Windows, Linux e Android através do navegador.
