# Requisitos Funcionais

## O que são?

**Requisitos funcionais** descrevem as funcionalidades e comportamentos que o sistema deve possuir para atender às necessidades dos usuários. Eles representam as ações, serviços e operações que a aplicação deve executar.

No Sistema Simplificado de Gestão Familiar, os requisitos funcionais estão relacionados ao controle financeiro e patrimonial, permitindo o gerenciamento de gastos, rendas e patrimônios de forma simples e organizada.

---

## Autenticação de Usuário

### RF01 – Cadastro de usuário
O sistema deve permitir o cadastro de usuários utilizando e-mail e senha.

### RF02 – Login no sistema
O sistema deve permitir que usuários autenticados realizem login utilizando e-mail e senha cadastrados.

---

## Gestão de Gastos

### RF03 – Cadastro de gastos
O sistema deve permitir a adição de gastos fixos e variáveis contendo:
- Nome do gasto;
- Valor;
- Setor/categoria.

### RF04 – Remoção de gastos
O sistema deve permitir a remoção de gastos cadastrados.

### RF05 – Visualização de gastos
O sistema deve apresentar uma tabela contendo todos os gastos cadastrados.

---

## Gestão de Rendas

### RF06 – Cadastro de renda
O sistema deve permitir a adição de rendas contendo:
- Nome da renda;
- Valor.

### RF07 – Remoção de renda
O sistema deve permitir a remoção de rendas cadastradas.

### RF08 – Visualização de rendas
O sistema deve apresentar uma tabela contendo todas as rendas cadastradas.

---

## Dashboard Financeiro

### RF09 – Exibição do dashboard financeiro
O sistema deve apresentar um dashboard financeiro com:
- Total de gastos;
- Total de rendas;
- Valor restante/saldo.

### RF10 – Filtro mensal
O dashboard deve permitir filtragem das informações por mês.

---

## Gestão Patrimonial

### RF11 – Cadastro de patrimônio
O sistema deve permitir a adição de patrimônios contendo:
- Nome do patrimônio;
- Valor do patrimônio.

### RF12 – Cálculo patrimonial
O sistema deve calcular automaticamente o valor total dos patrimônios cadastrados.
